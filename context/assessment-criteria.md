# Assessment Criteria

> This file defines how Claude should score incoming problems. Adjust weights and scales to match your team's priorities.

## Scoring Scale
Use a 1–5 scale for each dimension:
- **5** — Critical / Very High
- **4** — High
- **3** — Medium
- **2** — Low
- **1** — Negligible

## Dimensions

### 1. Business Impact
How much does solving this problem affect revenue, retention, or key business metrics?
- Consider: churn risk, revenue opportunity, operational cost reduction

### 2. User Experience Impact
How much does this problem degrade the experience of the users affected?
- Consider: frequency of the problem, severity, number of users affected

### 3. Strategy Alignment
How closely does this problem connect to current company strategy and product cycle priorities?
- Reference: @context/company-strategy.md and @context/product-cycle-strategy.md

## Output Format
For each dimension, provide:
- Score (1–5)
- One-sentence rationale
- Overall recommendation: **Prioritize / Investigate / Backlog / Reject**
